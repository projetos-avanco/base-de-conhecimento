* LINKAR PORTA SERIAL
  Link Simbólico(Transformar uma placa serial, por exemplo, de "ttyF" para ttyS, Linux):

-> Editar o arquivo rc.local:
mcedit /etc/rc.d/rc.local

-> Incluir em qualquer local do rc.local:
rm -rf /dev/ttyS0
ln -s /dev/ttyF0 /dev/ttyS0

-> Logo em seguida, reiniciar a máquina do caixa

-> Digitar esse comando para ver se subiu corretamente:
ls -la /dev/ttyS0
setserial -bg /dev/ttyS*


* Adptação postgres
Se caso for adaptar o banco postgres e ele não criar as devidas tabelas ou der erro, deverá fazer os seguintes procedimento:
	- Fazer backup do postgres (Backup da pasta 'data' e pelo aplicativo do postgres)
	- Após feito o backup, deverá desinstalar o Postgres: 1º: Pelo Painel de Controle; 2º: Removendo o usuario postgres do gerenciamento; 3º: Deletar a pasta c:\Arquivos de programas\PostgresSQL\8.3
	- Reiniciar a máquina
	- Instalar o Postgres da versão(Lei da Transparencia) xx.02.40 e logo em seguida adaptar para versão que vai ser atualizada.
	- Restaurar apenas "DADOS" do arquivo xxxx.backup feito anteriormente no novo banco
	- Reiniciar a máquina, novamente
	- Verificar se criou as tabelas e colunas, e se foi restaurado corretamente os dados

* Mensagem ao executar o comando psql -U postgres
	psql: could not connect to server: No such file or directory
        Is the server running locally and accepting
        connections on Unix domain socket "/tmp/.s.PGSQL.5432"?

	Conecte no servidor
	- cd /u
	- chmod -R 700 pgsql && chown -R postgres.postgres pgsql
	- sh /etc/rc.d/rc.postgresql start
	- psql -U postgres
	-** se nao levantar
		- apaga o arquivo /u/pgsql/serverlog
		- tenta dar o start no postgres de novo
		- ele vai criar o serverlog novamente
		- mais agora ele vai estar te informando o motivo de ele nao ter iniciado

* Postgres no Linux não inicia(Stoping PostgreSQL: failled, Starting PostgreSQL: ok)
	- acessar o servidor com super usuario
	- cd /u/pgsql/
	- rm serverlog
	- depois tentar iniciar o serviço: sh /etc/rc.d/rc.postgresql start
	- verificar se iniciou: sh /etc/rc.d/rc.postgresql status, caso apareça: pg_ctl: no server running, verificar o arquivo serverlog
	- more serverlog
	- se caso no log estiver escrito "FATAL: bogus data in lock file "/tmp/.s.PGSQL.5432.lock": "" ", fazer rm /tmp/.s.PGSQL.5432.lock
	- depois só iniciar o serviço: sh /etc/rc.d/rc.postgresql start
